# Architect Review Loop

Review against:
- requirements coverage
- acceptance evidence
- regression risk
- missing tests
- migration completeness

Loop:
1. review the current diff or output
2. identify highest-risk issues first
3. fix or delegate fixes
4. rerun verification
5. re-review

Do not stop at summary only.
End with:
- pass/fail
- remaining risks
- follow-up work if needed
